間ma · the space between

The idea

A broadsheet is loud. A haiku is not.

A small computer — most likely one you already own — sits on the shelf with a panel of electronic paper and no controls at all. On a schedule you set — say seven each morning — it gathers today's headlines, weighs the mood of the day, and writes one original haiku, three lines, 5·7·5, that catches not the headlines but the feeling of the morning. It prints the verse in black with a seal in red, and it's simply there when you come down — the way the paper used to be.

That's the whole promise: the day's noise arrives, and something quieter is left in its place.

再sai · again, anew

What it takes

A second life for a board you already have

The first plan called for a brand-new Raspberry Pi Zero 2 W. Then they sold out everywhere — and besides, the least wasteful machine is the one already in the drawer. A Raspberry Pi 3 Model A+, boxed since 2020, is gloriously over-qualified for an errand it runs once a morning. So this build reuses what you have. The one thing you buy is the page.

勿体無 mottainai · what a waste

There's a word in Japanese for the small grief of letting a usable thing go to waste — mottainai. And a craft, kintsugi, that mends a broken bowl with gold so its history is shown rather than hidden, the repair becoming the most honest part of the object.

Hokku is neither broken nor mended. But it runs on the same instinct: a capable little machine, dark for five years, given a quiet daily purpose instead of a landfill.

流nagare · the flow

How a morning goes

Five quiet steps, once a day, then silence

  1. The clock wakes it

    A single cron line fires at the hour you chose. There's no daemon humming all day — the script runs, finishes, and the board goes back to idle.

  2. It gathers the headlines

    Three RSS feeds — BBC News, Sky News, The Guardian — parsed with nothing heavier than the standard library's xml.etree. A handful of titles from each, no scraping, no extra dependency.

  3. It asks Claude to compose

    The headlines are pasted into the prompt as plain text and sent to the Messages API as a single flat-rate completion — no tools, no browsing. Claude composes from what it's handed.

  4. Three lines come back

    The response is parsed for the text blocks; out falls the morning's haiku.

  5. The pHAT inks it

    The verse is laid out in black, the seal stamped in red with a small date beside it — so a stale morning reads as stale, not as today's — and pushed to the e-paper. A few seconds of refresh, and it's there until tomorrow.

機ki · the mechanism

Under the hood

It clicks together, no iron required

The 3A+ ships with its 40-pin header already soldered, so the Inky pHAT seats onto it and the device is complete. The panel only borrows a handful of those forty lines; the rest are along for the ride.

40-pin header — the pHAT seats on all of it, but only a few lines do any work: SPI GPIO10 / GPIO11 / GPIO8 → pixels to the panel GPIO RESET 27 · BUSY 17 · DC 22 → the refresh handshake I²C GPIO2 / GPIO3 → reads the pHAT's EEPROM, IDs the panel variant PWR 3V3 · 5V · GND → power No jumper wires. No breadboard. It just presses down.
  • BoardRaspberry Pi 3 Model A+ — quad-core, onboard Wi-Fi, 40-pin header.
  • PageInky pHAT — 250×122, four-colour e-paper (EEPROM variant 23, InkyJD79661 driver).
  • FeedRSS — BBC News, Sky News, The Guardian. Parsed with the standard library, no scraping.
  • BrainClaude Opus 4.8 · Messages API — one flat-rate completion, no tools. The restraint is in max_tokens, not the model — the same instinct as the board: full capability, deliberately given a small job.
  • Clockcron — one wake a day, then the board sleeps.
  • Keys/etc/hokku.env — the API key, sourced at run, never in the repo.

The one-shot

Read the morning, print a haiku, exit

No framework, no agent loop on the board — a clean script that runs top to bottom and stops. Three small pieces: the key, the script, the schedule.

hokku.py — gather, compose, ink, exit 
#!/usr/bin/env python3
import os, json
import urllib.request
import xml.etree.ElementTree as ET
from inky.auto import auto

FEEDS = {
    "BBC News":     "http://feeds.bbci.co.uk/news/rss.xml",
    "Sky News":     "https://feeds.skynews.com/feeds/rss/home.xml",
    "The Guardian": "https://www.theguardian.com/uk/rss",
}

def fetch_headlines(limit=8):
    titles = []
    for name, url in FEEDS.items():
        req = urllib.request.Request(url, headers={"User-Agent": "hokku/1.0"})
        with urllib.request.urlopen(req, timeout=10) as resp:
            root = ET.fromstring(resp.read())
        titles += [item.findtext("title") for item in root.iter("item")][:limit]
    return [t for t in titles if t]

headlines = fetch_headlines()

prompt = (
    "Here are today's UK headlines:\n"
    + "\n".join("- " + h for h in headlines)
    + "\n\nWrite ONE original haiku, three lines 5-7-5, that catches "
      "the mood of the morning — not any single headline. "
      "Reply with the three lines only."
)

body = json.dumps({
    "model": os.environ.get("HOKKU_MODEL", "claude-opus-4-8"),
    "max_tokens": 300,           # the restraint is here, not in the model
    "messages": [{"role": "user", "content": prompt}],
}).encode()

req = urllib.request.Request(
    "https://api.anthropic.com/v1/messages",
    data=body,
    headers={
        "x-api-key": os.environ["ANTHROPIC_API_KEY"],
        "anthropic-version": "2023-06-01",
        "content-type": "application/json",
    },
)
with urllib.request.urlopen(req, timeout=30) as resp:
    payload = json.loads(resp.read())
text = "".join(b["text"] for b in payload["content"] if b["type"] == "text")
lines = [l for l in text.strip().splitlines() if l.strip()]
haiku = "\n".join(lines[-3:])   # last 3 non-empty lines — Claude sometimes adds a preamble

# ---- ink it: palette-mode image, black text, red seal, then refresh ----
display = auto()
img = render(haiku, display.width, display.height)   # render(): your layout — see repo
display.set_image(img)
display.show()
/etc/cron.d/hokku — 07:00, every day 
# wake · gather · compose · ink — source the env, then run once
0 7 * * *  pi  /usr/bin/env bash -c 'set -a; . /etc/hokku.env; set +a; \

注chū · take note

Honest notes

What to expect, plainly

It needs the network

Onboard Wi-Fi must be in range at the appointed hour, for both the RSS pulls and the API call. If a morning's request fails, the script exits before it ever touches the panel — no partial render, no error screen, just a line in the log. The e-paper only draws current to change, so a failed morning costs nothing and simply leaves yesterday's verse up a day longer.

It asks, it doesn't browse

An earlier version let Claude fetch the front pages itself, agentic search switched on. It read beautifully and cost real money — about £6 in a single day of testing, billed per search. Headlines pasted in as plain context write just as well, and a single flat-rate completion costs the same every morning: six pounds a day became a fraction of a penny a morning.

Old Pi, old Python

System Python on this board is 3.7, which the e-paper library's newer GPIO dependency doesn't support — so hokku.py carries a small compatibility shim. It works; the only visible trace is a harmless warning some mornings while the display finishes refreshing.

E-paper is slow, and that's the point

A refresh takes a few seconds and flickers as it settles — which for a thing you read once at breakfast is exactly right.

Every morning is new

The haiku is generated fresh each day, so it's never the same twice and never quite predictable. Treat that as the feature it is.

紙

The ink that stays

Electronic paper holds its image with the power pulled. The verse you read at breakfast is still there at dusk, and would still be there if the plug came out — the panel only draws current to change, never to hold.

Nothing is spent keeping the morning in view. It is the most patient screen ever made, and close to the least wasteful — the nearest a display comes to actual ink on actual paper, sitting on a board that itself refused to be thrown away.
